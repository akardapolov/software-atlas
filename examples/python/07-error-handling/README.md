# Error Handling in Python

## Overview

Python provides **exceptions** for error handling, **context managers** for
 resource management, and **assertions** for validation. The `logging` module provides structured output.

## How to Run

```bash
python3 main.py
```

## Key Concepts

- **Exceptions** — `try/except/else/finally` blocks for error handling
- **Built-in exceptions** — `Exception`, `ValueError`, `KeyError`, etc.
- **Custom exceptions** — inherit from `Exception`
- **Logging** — `logging` module for structured output
- **Assertions** — `assert` for validation and invariants
- **Context managers** — `contextlib` for resource management

## Historical Context

Python's error handling evolved from simple `try/except` to include
context managers, logging, and assertions. This provides robust error handling for production applications.
