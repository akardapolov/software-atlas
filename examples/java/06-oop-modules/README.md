# OOP and Modules in Java

## Overview

Java supports **class-based OOP** with single inheritance, **interfaces**
for polymorphism, **abstract classes**, and **packages**. The language
emphasises type safety with strong typing and compile-time checking.

## How to Run

```bash
# Compile
javac Main.java

# Run
java com.example.Main
```

## Key Concepts

- **Classes** — `class` keyword, `extends` for inheritance
- **Interfaces** — `interface` for contracts, `implements` keyword
- **Abstract classes** — `abstract class`, `@abstractmethod`
- **Access modifiers** — `public`, `protected`, `private`, package-private
- **Static** — `static` members, `@staticmethod`, `@classmethod`
- **Final** — `final` class/method (cannot be overridden/extended)
- **Generics** — `<T>` type parameters for type safety
- **Inner classes** — static and non-static nested classes
- **Enums** — `enum` for enumerations with values and methods
- **Exceptions** — `try-catch-finally`, custom exceptions
- **Method overloading** — same name, different parameters
- **Anonymous classes** — implementing interfaces without naming

## Historical Context

Java (1995, Sun) popularised OOP with garbage collection. Classes are single-inheritance only (unlike C++). Interfaces enable decoupling and multiple inheritance of types. Generics (Java 5, 2004) added type-safe collections and algorithms.
