## Summary

Newman systematized microservices as an architectural style:
small autonomous services, independent deployments, data ownership,
observability, and automation.

The book is valuable because it constantly returns to trade-offs:
microservices buy independence at the cost of distributed system complexity.

## Key Ideas

- Decomposition: bounded contexts as natural boundaries
- Data ownership: "database per service" → eventual consistency
- Integration: synchronous calls vs asynchronous events
- Operational excellence: monitoring, logging, tracing
- Deployment: automation, pipelines, safe releases
