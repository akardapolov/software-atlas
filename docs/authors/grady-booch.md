## Biography

Grady Booch is an American software engineer and one of the pioneers of **object-oriented analysis and design (OOAD)**. His **Booch Method** was one of the first systematic approaches to object-oriented modeling, predating UML.

Booch, along with James Rumbaugh and Ivar Jacobson (the "three amigos"), co-created the **Unified Modeling Language (UML)** in the mid-1990s, which became the industry standard for visualizing software design.

## Key Contributions

### Booch Method

Booch's OOAD methodology included:

- **Object-Oriented Analysis and Design** — systematic approach to OO modeling
- **Booch notation** — one of the first graphical notations for OO:
  - Cloud symbols for classes (later replaced by rectangles)
  - Different arrow types for relationships
- **Classification of relationships**:
  - **Association** — basic relationship between objects
  - **Aggregation** — "has-a" relationship (whole-part)
  - **Composition** — strong aggregation (parts don't exist independently)
  - **Dependency** — one object depends on another
  - **Inheritance** — "is-a" relationship

### Unified Modeling Language (UML)

In 1995, Booch, Rumbaugh, and Jacobson merged their methodologies into UML:

- **UML 1.0 (1997)** — first standard version
- **Unified notations** — single standard for all OO diagrams
- **Industry adoption** — tools (Rational Rose, Enterprise Architect), education, documentation

UML's core diagrams:
- **Class diagrams** — object structure
- **Use case diagrams** — Jacobson's contribution
- **Sequence diagrams** — interaction over time
- **State diagrams** — Rumbaugh's contribution

### Software Architecture

Booch contributed to software architecture thinking:
- **Architectural patterns** — layering, microkernel, pipes and filters
- **Architecture evaluation** — ATAM (Architecture Tradeoff Analysis Method)
- **Books** — "Software Architecture in Practice" co-authored

## Influence

### Influenced by

- **Simula / Smalltalk** — object-oriented modeling concepts
- **Entity-Relationship modeling** — Chen's ER diagrams

### Influenced

- **UML standard** — Booch notation became part of UML class diagrams
- **Software architecture** — architectural thinking and patterns
- **Rational Unified Process (RUP)** — methodology built on Booch's work

## Legacy

Booch's impact:
- **UML became universal language** for software design
- **OOAD became teachable** — systematic approach to OOP design
- **Tool ecosystem** — Rational Rose, Enterprise Architect, and modern UML tools
