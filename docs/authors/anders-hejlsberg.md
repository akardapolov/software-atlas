## Overview

Anders Hejlsberg is a Danish software engineer and the lead architect behind three of the most influential programming languages in modern software development: **Turbo Pascal**, **Delphi**, and **C#**. He is also the creator of **TypeScript**.

Hejlsberg's career has been defined by a consistent vision: building languages that combine power with productivity, type safety with flexibility, and performance with developer ergonomics.

---

## Key Contributions

### Turbo Pascal (1983)

Hejlsberg's first major language. Turbo Pascal was an integrated development environment and compiler that made Pascal accessible and fast on early PCs. It became the standard teaching language and professional development tool for DOS-era programming.

### Delphi (1995)

At Borland, Hejlsberg created Delphi — a rapid application development (RAD) environment combining Object Pascal with a visual component framework. Key innovations:
- **Properties** — first-class language feature replacing getters/setters
- **Events** — delegate-like callback mechanism
- **Component model** — visual design-time programming
- **Fast native compilation** — ahead-of-time compilation to native code

These concepts directly influenced C#.

### C# (2000–present)

As lead language designer at Microsoft, Hejlsberg created C# as part of the .NET initiative. Released in 2002, C# has evolved through 13 major versions, consistently adding modern features:
- Generics, LINQ, async/await, nullable reference types, records, pattern matching

Hejlsberg continues to lead C# language design today.

### TypeScript (2012–present)

Hejlsberg created TypeScript to bring static typing and modern language features to JavaScript. TypeScript has become the de facto standard for large-scale JavaScript development.

## Design Philosophy

Hejlsberg's language design is guided by several principles:

1. **Pragmatism over purity** — borrow good ideas from many sources, make them accessible
2. **Tooling matters** — the language is designed alongside the IDE that supports it
3. **Evolution without breakage** — add features compatibly, don't break existing code
4. **Type safety progressively** — make the type system powerful but not restrictive
5. **Features compose** — individual features should work well together

## Influence

### On Languages

| Language | Influence |
|----------|-----------|
| **C#** | Properties, events, delegates, LINQ, async/await, records |
| **TypeScript** | Structural typing, gradual typing, modern JavaScript features |
| **Kotlin** | Properties, data classes, null safety, extension methods (inspired by C#/Delphi) |
| **Dart** | Similar syntax, properties, async/await |

### On Platform Design

- **.NET CLR** — managed runtime with JIT compilation, GC, and language interoperability
- **Roslyn** — open-source compiler platform enabling analyzers and source generators
- **Language Server Protocol adoption** — TypeScript's success drove LSP standardization
