## Biography

Erich Gamma is a Swiss computer scientist and one of the **Gang of Four** authors of the landmark *"Design Patterns: Elements of Reusable Object-Oriented Software"* (1994). He also co-created **JUnit** with Kent Beck and served as chief architect for the **Eclipse IDE**.

Gamma's work spans theoretical patterns (GoF), practical testing (JUnit), and large-scale systems (Eclipse, Microsoft Visual Studio Code).

## Key Contributions

### Gang of Four (GoF) Design Patterns

Gamma, with Richard Helm, Ralph Johnson, and John Vlissides, created the definitive catalog of 23 design patterns:

- **Creational patterns** — Factory, Builder, Prototype, Singleton
- **Structural patterns** — Adapter, Decorator, Facade, Proxy, etc.
- **Behavioral patterns** — Command, Observer, Strategy, Template Method, etc.

The book provided:
- **Shared vocabulary** — "this is a Factory pattern" is understood across teams
- **Trade-off thinking** — patterns discuss consequences and alternatives
- **Reusable solutions** — patterns extracted from practical experience

### JUnit

With Kent Beck, Gamma co-created JUnit:

- **xUnit framework** — JUnit inspired NUnit, PHPUnit, etc.
- **Test-driven development support** — simple, practical unit testing
- **Open source** — widely adopted testing framework

### Eclipse Architecture

As Eclipse's chief architect, Gamma designed:

- **Plugin architecture** — extensible IDE platform
- **Rich Client Platform (RCP)** — application framework
- **Architecture patterns** — implementation patterns at framework scale

### Influence on Frameworks

Gamma's work influenced:
- **Spring Framework** — heavily uses GoF patterns
- **.NET libraries** — patterns in base class library
- **Modern frameworks** — React, Angular, AngularJS use patterns
