## Biography

Mike Cohn is a consultant, trainer, and author known for *User Stories Applied* (2004) and co-author of *User Stories Applied for Agile Teams* (2006). Cohn is widely recognized as an authority on user stories and their application in agile development and product management.

Cohn's work focuses on making user stories more practical and actionable, helping teams deliver customer value more effectively through better collaboration and estimation practices.

## Key Contributions

### User Stories (2004)

Cohn established the **INVEST criteria** for evaluating user story quality:

| Criterion | Meaning | Good Example | Bad Example |
|-----------|---------|-------------|------------|
| **Independent** | Can be implemented separately | "As a user, I want to reset my password" | "As a user, I want to reset my password and change my profile picture" |
| **Negotiable** | Details can be discussed | "Checkout with credit card" | "Checkout with PayPal and saved cards" |
| **Valuable** | Delivers customer value | "Search for products" | "Search products and see ads" |
| **Estimable** | Team can estimate effort | "Add product to cart" | "Redesign entire checkout flow" |
| **Small** | Can be completed in one iteration | "Add product to cart" | "Redesign entire checkout flow" |
| **Testable** | Can verify implementation | "Checkout succeeds" | "Checkout succeeds if payment gateway is responsive" |

**Key practices:**
- Write stories from user's perspective
- Include acceptance criteria
- Keep stories small and focused
- Use "As a..." format

### User Stories Applied for Agile Teams (2006)

With David Hussman and Kent Beck, Cohn expanded on applying user stories in team contexts:

**Team collaboration:**
- Story writing workshops with stakeholders
- Collective estimation (planning poker)
- Story mapping and journey visualization
- Regular backlog refinement

**Story breakdown:**
- Epics → Stories → Tasks
- Split large stories into manageable chunks
- Define clear definition of done

### Agile Estimation Practices

Cohn contributed to estimation techniques:

| Technique | Description | When to Use |
|-----------|-------------|-------------|
| **Planning Poker** | Consensus-based estimation | Estimating effort for upcoming sprint |
| **Story Points** | Relative complexity measure | Abstract away hours, focus on complexity |
| **T-shirt Sizing** | Simple, intuitive | Quick estimation for small items |
| **Velocity Tracking** | Measure team output | Predict future capacity |
