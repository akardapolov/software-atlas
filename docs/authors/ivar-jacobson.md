## Biography

Ivar Jacobson is a Swedish computer scientist and one of the pioneers of **object-oriented software engineering**. While working at Ericsson on the AXE telecom switch system, he developed **Use Cases** as a way to model requirements from the user's perspective.

Jacobson, with Grady Booch and James Rumbaugh, co-created **UML (Unified Modeling Language)**, contributing the **use case diagram** notation that became part of the standard.

## Key Contributions

### Use Cases

Jacobson introduced **Use Cases** as a scenario-based approach to requirements:

- **Actor** — external entity interacting with the system
- **Use case** — sequence of interactions between actor and system to achieve a goal
- **Storytelling format** — natural language description of behavior

Use cases became the primary way to:
- **Elicit requirements** — talk to users in their language
- **Specify behavior** — define what system should do, not how
- **Drive testing** — test scenarios mapped to use cases

### OOSE Methodology

**Object-Oriented Software Engineering (OOSE)** was Jacobson's comprehensive methodology:

- **Use-case driven design** — start from user scenarios
- **Domain modeling** — objects representing domain concepts
- **Traceability** — trace from requirements through design to code
- **Interfaces as contracts** — use cases define interfaces before implementation

### UML Use Case Diagrams

Jacobson's use case concept became a core UML diagram type:

```
[Customer] -- (Browse Products)
[Customer] -- (Add to Cart)
[Customer] -- (Checkout)
[Customer] -- (Make Payment)
```

Use case diagrams show:
- **Actors** — stick figure with name
- **Use cases** — ovals with descriptions
- **Relationships** — include, extend, generalization

### Influence on Agile

Use cases directly influenced **Agile methodologies**:
- **User Stories** — simplified use cases (INVEST criteria)
- **Acceptance criteria** — use case scenarios as testable conditions
- **Behavior-driven development** — Cucumber scenarios evolved from use case thinking

## Influence

### Influenced by

- **Ericsson AXE project** — real-world telecom system where use cases were developed
- **Object-oriented thinking** — Simula, Smalltalk methodologies

### Influenced

- **UML standard** — use case diagrams became core UML artifact
- **Requirements engineering** — use cases became standard practice
- **Agile methodologies** — user stories evolved from use cases
- **Alistair Cockburn** — extended use case thinking in "Writing Effective Use Cases"

## Legacy

Jacobson's impact:
- **User-centric design** — requirements from user perspective
- **Standardized communication** — use cases bridge business and technical
- **UML foundation** — part of universal modeling language
