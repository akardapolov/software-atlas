## Biography

Bjarne Stroustrup is a Danish computer scientist and the creator of **C++**. Starting with "C with Classes" in 1979, he built a language that brought **Simula-style object-oriented programming** to the world of **systems programming**.

Stroustrup's goal was "zero-overhead abstraction" — features should cost nothing if not used. This principle, combined with C's efficiency and Simula's abstractions, made C++ the dominant systems language of the 1990s and 2000s.

## Key Contributions

### C++ Design Principles

Stroustrup designed C++ with several key innovations:

- **Classes with zero abstraction cost** — you pay only for what you use
- **Multiple inheritance** — powerful, but introduces the diamond problem
- **Templates** — compile-time parametric polymorphism → Standard Template Library (STL)
- **RAII** (Resource Acquisition Is Initialization) — automatic resource management via destructors
- **Operator overloading** — objects behave like built-in types
- **References** — safe alternatives to pointers

### Making OOP Mainstream

C++ was the first language to make OOP mainstream in industry:
- Systems programming with OOP became possible
- Syntax influenced Java, C#, D, Rust
- Standard Template Library popularized generic programming

### Design and Evolution Philosophy

Stroustrup emphasized:

- **Evolution, not revolution** — build incrementally on C
- **You don't pay for what you don't use** — zero-overhead principle
- **Multi-paradigm** — procedural, OOP, generic programming all coexist
- **Standardization** — ISO C++ standard ensures compatibility

## Influence

### Influenced by

- **Ole-Johan Dahl** — Simula 67 (classes, inheritance)
- **C language** — efficiency, low-level control, systems programming

### Influenced

- **Mainstream OOP languages** — Java, C#, D, Rust (syntax inheritance)
- **Generic programming** — STL → Java Generics, C# generics, Rust traits
- **RAII patterns** → Rust ownership model, smart pointers in C++
