# Simon Brown

## Biography

Simon Brown is a software architect and author known for creating the **C4 Model** — a hierarchical approach to diagramming software systems. He has over 20 years of experience in software architecture and has worked with companies across various industries.

Brown's work focuses on making software architecture more accessible and practical through simple, effective diagramming techniques that complement the traditional UML approach.

## Key Contributions

### The C4 Model (2011+)

Brown created the C4 Model as an alternative to complex UML diagrams. C4 provides four levels of abstraction:

| Level | Description | Audience |
|--------|-------------|----------|
| **Level 1: Context** | High-level overview showing system in its environment | Everyone |
| **Level 2: Containers** | Applications, databases, and services | Technical leadership |
| **Level 3: Components** | Internal structure of containers | Developers |
| **Level 4: Code** | Class diagram (optional, often auto-generated) | Developers |

**Why C4 matters:**
- UML can be overly complex and intimidating for non-architects
- C4 focuses on communication over completeness
- Simple notation can be drawn quickly and understood by all stakeholders
- Levels match different communication needs

### Diagramming Philosophy

Brown advocates for:
- **Abstraction at appropriate levels** — don't show everything to everyone
- **Simple is better** — hand-drawn sketches over complex tools
- **Code as documentation** — diagrams should be kept with the code
- **Iterative refinement** — start rough, refine as needed

### Architecture Principles

From his work, Brown promotes:
- **Communication over documentation** — diagrams facilitate discussion
- **Visual clarity** — simple, clear diagrams over comprehensive ones
- **Tool independence** — whiteboards, marker pens, simple drawing tools
- **Collaboration** — live diagramming sessions
