# Topics

Knowledge organized by domain. Each topic traces how ideas evolved
from foundational concepts to modern practice.

## Topic Areas

| # | Topic | Description | Key ideas |
|---|-------|-------------|-----------|
| 1 | [Paradigms](paradigms/) | How we think about computation | Imperative, OOP, FP, Declarative |
| 2 | [Architecture & Modularity](architecture/) | How we structure systems | Layered, Hexagonal, DDD, Microservices |
| 3 | [OOP & Design](design/) | How we design components | SOLID, GoF patterns, Refactoring |
| 4 | [Type Systems](types/) | How types help us reason | Static/dynamic, ADTs, generics |
| 5 | [Functional Programming](functional/) | How we avoid accidental complexity | Purity, immutability, composition |
| 6 | [Concurrency](concurrency/) | How we handle parallelism | Threads, CSP, Actors, async |
| 7 | [Process & Testing](process/) | How we build and ship | Agile, TDD, CI/CD, DevOps |
| 8 | [Version Control & Git](vcs/) | How we track and evolve code | Git, branching, workflows |
| 9 | [Distributed Systems](distributed/) | How we scale across machines | Clocks, CAP, consensus, streaming |

## Cross-cutting Themes

Many ideas appear across multiple topics:

- **Modularity** touches architecture, design, and FP
- **Types** connect to design, FP, and languages
- **Testing** relates to process, design, and architecture
- **Simplicity** is a recurring theme from Dijkstra to Hickey
