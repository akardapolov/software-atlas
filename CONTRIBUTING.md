# Contributing to Software Atlas

Thank you for your interest in contributing to Software Atlas!

## About This Project

Software Atlas is primarily a **personal knowledge base** documenting the evolution
of software development ideas, practices, and technologies. It serves as a
reference for understanding how concepts connect and influence each other over time.

## How to Contribute

### Bug Reports and Corrections

If you find factual errors, broken links, or incorrect information:
1. Open an issue describing the problem
2. Provide sources for corrections when possible

### Content Additions

Suggestions for new topics, authors, works, or connections are welcome. Please:
1. Check that the content isn't already covered
2. Open an issue to discuss before submitting
3. Follow the existing structure and format

### Code Examples

New code examples should:
- Follow the existing numbering scheme (01, 02, 03, etc.)
- Include a README.md explaining the example
- Be idiomatic for the language
- Be well-commented

## Guidelines

### Content
- Focus on historical accuracy and clear explanations
- Provide sources for claims
- Use existing templates and formats
- Keep descriptions concise but informative

### Formatting
- Use Markdown for documentation
- Follow existing file structure
- Use proper heading hierarchy
- Include relevant links between related topics

### Code
- Keep examples simple and focused
- Avoid external dependencies when possible
- Include compilation/run instructions in README

## Development

The site is built with MkDocs and the Material theme.

```bash
# Install dependencies
pip install -r requirements.txt

# Serve locally
mkdocs serve

# Build for deployment
mkdocs build
```
